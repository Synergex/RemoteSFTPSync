SFTP Sync is a utility application that can synchronize a local directory structure and files to a remote OpenVMS system via the Secure FTP protocol.

 

When first activated the product will first check whether the local and remote directories are synchronized, and if not will update the remote directories and files to match the local directories and files. This includes deleting any remote files that no longer exist in the local directories.

 

Once synchronized the application then monitors the local directories for changes, and replicates those changes in the remote system.

 

Two versions of the application are available, one with a Windows user interface, and one used from the Windows command line.

 

Important Usage Information

 

Regardless of whether you use the command-line or Windows desktop application, it is important that you understand how the product works, so please read the information below carefully.

 

When you first start the command-line product, or enable sync in the Windows desktop tool, a worker thread is created to take care of syncing each of the file spec patterns that you have specified. So if you have specified four file extensions, like this:

 

*.DBL;*.DEF;*.COM;*.SCM

 

Then multiple worker threads are started, one to handle each file extension pattern.

 

Before any files can be synced the entire directory tree must be present on the remote system, and ensuring that this is the case is a task assigned to the first worker thread to be created. Once that task has been completed, that thread can then begin the task of checking whether the files on the remote system currently match the files on the local system, and uploading any missing files for that file extension.

 

As soon as the first worker thread has completed it's additional task of checking the directory structure, any other worker threads will be created and will go to work performing the initial synchronization based on their assigned file extension pattern.

 

As soon as each worker thread has completed its initial sync, it starts monitoring the file system for changes to files matching the assigned file extension pattern and replicating those changes to the remote system.

 

When real time synchronization commences you will see a message Initial sync complete, real-time sync now active. You should not make changes to files in your local directories tree until you have seen this message.
